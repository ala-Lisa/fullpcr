# 分析参数常驻显示设计

## 目标

分析工作台中的“分析参数”不再使用外层折叠框。基础参数在页面中始终直接显示，减少一次无必要的展开操作。

## 界面行为

- 删除标题为“分析参数”的 `st.expander`。
- 在原位置直接显示“分析参数”标题、默认参数说明和现有基础参数控件。
- 保留“显示高级参数”开关；高级参数仍默认隐藏，开启后才显示。
- 保留“显示高级分步工作流”开关及其现有行为。

## 状态与命令

本次只改变外层布局容器，不修改控件 key、默认值、`session_state`、参数构建逻辑或最终执行命令。

## 测试与验收

- AppTest 断言页面不存在标题为“分析参数”的 expander。
- 基础参数控件默认可见。
- 高级参数默认隐藏，开启“显示高级参数”后可见。
- 运行相关 GUI 测试、全量测试、编译检查和 `git diff --check`。

## 范围外

不调整参数数值、帮助文字、命令语义、输入验证、结果展示或其他页面布局。
