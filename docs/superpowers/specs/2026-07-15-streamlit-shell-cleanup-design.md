# Streamlit 顶栏与侧栏滚动清理设计

## 目标

- 移除页面顶部由 Streamlit `stHeader` 产生的半透明空白区域。
- 桌面端侧栏不出现独立纵向滚动条。
- 保留现有品牌卡、环境检查、三项导航和内部平台信息。
- 不改变页面路由、分析流程、结果展示或下载行为。

## 现状与原因

- 顶部空白来自 `[data-testid="stHeader"]`。当前只隐藏了工具栏和菜单，仍保留顶栏高度和半透明背景。
- 侧栏内容总高度超过视口后，`stSidebarContent` 自动产生纵向滚动条。

## 设计

### 顶栏

将 `stHeader` 的高度和最小高度设为 `0`，取消背景、阴影和模糊效果，并禁止其接收鼠标事件。继续隐藏工具栏、菜单、Deploy 按钮和装饰条。主内容从现有 `.block-container` 顶部内边距开始，不增加新的占位元素。

### 侧栏

让 `stSidebarContent` 使用一屏高度的纵向弹性布局并隐藏自身溢出。保持以下顺序：

1. 品牌卡
2. 环境状态
3. 分析导航
4. 内部平台信息

底部信息通过自动上边距贴近侧栏底部。缩小品牌卡、各区块间距和导航卡纵向尺寸，但不缩小到影响标题与说明文字可读性。

针对较矮的桌面视口增加 `max-height` 媒体查询，进一步压缩空白和内边距，确保主要内容仍在一屏内。侧栏不使用缩放变换，也不隐藏任何功能入口。

## 兼容性与边界

- 保持 `initial_sidebar_state="locked"`，侧栏仍不可折叠丢失。
- 只针对桌面侧栏清理滚动；不改变主内容区域的正常页面滚动。
- 不修改 Streamlit session state、导航选项文字或环境检查缓存。
- 如果极端小高度窗口无法容纳全部内容，优先保持功能可见，不通过删除模块解决。

## 验证

- 增加样式契约测试，确认 `stHeader` 高度为零且侧栏内容隐藏独立溢出。
- 运行 GUI 测试和完整 `pytest -q`。
- 运行 `python3 -m compileall -q fullpcr` 与 `git diff --check`。
- 在浏览器 125% 缩放下截图检查：顶部无空白玻璃栏、侧栏无滚动条、四个侧栏区域均完整显示。
- 重启本地服务并确认 `http://localhost:18503/` 返回 HTTP 200，健康检查返回 `ok`。
