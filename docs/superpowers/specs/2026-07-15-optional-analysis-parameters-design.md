# 顶部分析参数交互整理设计

## 目标

- 将工作台顶部 `分析参数（可选）` 改为 `分析参数`。
- 明确“不修改即使用默认参数”，不再用“可选”描述整个区域。
- 基础参数保持直接可编辑；高级参数只有勾选后才显示数值输入并覆盖默认行为。
- 为每种数值设置符合参数语义的加减步长。
- 不修改五步手工工作流中的参数区域和核心分析算法。

## 参数分组

### 基础参数

以下参数继续直接显示，并由当前预设提供初始值：

- 最小扩增片段长度
- 最大扩增片段长度
- 允许错配数
- obipcr mismatch 等级
- 环状序列

用户不修改时沿用当前预设值。数值步长为：扩增长度 `10 bp`，允许错配数 `1`。

### 高级参数

以下参数使用“启用复选框 + 条件数值输入框”：

| 参数 | 未启用时 | 启用后初始值 | 步长 |
|---|---|---:|---:|
| 最低 Tm | 使用 fullpcr 当前默认 `50°C` | `50.0` | `0.5°C` |
| misStart | 不传 CLI flag，使用 MFEprimer 默认 `1` | `1` | `1` |
| misEnd | 不传 CLI flag，使用 MFEprimer 默认 `9` | `9` | `1` |
| mono | 不传 CLI flag，使用 MFEprimer 默认 `50 mM` | `50.0` | `1.0 mM` |
| diva | 不传 CLI flag，使用 MFEprimer 默认 `1.5 mM` | `1.5` | `0.1 mM` |
| dNTP | 不传 CLI flag，使用 MFEprimer 默认 `0.25 mM` | `0.25` | `0.05 mM` |
| oligo | 不传 CLI flag，使用 MFEprimer 默认 `50 nM` | `50.0` | `1.0 nM` |

`输出引物结合位点` 和 `从扩增序列中切除引物` 本身就是布尔 CLI flag，继续使用单个复选框。选中时加入命令，未选中时不加入。

## 交互规则

- 高级参数默认全部未启用，只显示各自的启用复选框。
- 选中某项后，在该项下方显示对应数值输入框。
- 取消选中后，命令立即恢复该参数的默认行为。
- 取消选中不清空已经输入的数值；再次启用时恢复上次输入值。
- 应用引物类型预设只修改现有基础参数，不启用、禁用或覆盖高级参数。
- 页面切换、输入文件验证和高级工作流开关不得清除高级参数启用状态或已填写数值。

## 状态与命令契约

- 为七个数值高级参数增加独立的 canonical 启用状态，确保跨页面保持。
- `common_params` 以启用状态决定返回覆盖值还是默认值/`None`，不能仅依据隐藏输入框中残留的数值。
- 最低 Tm 未启用时继续返回 `50.0`，保持现有 fullpcr 命令行为。
- `misStart`、`misEnd`、`mono`、`diva`、`dNTP`、`oligo` 未启用时返回 `None`，由命令构建器省略对应 CLI flag。
- 一键完整分析和高级步骤 3 继续读取同一份 `common_params`，两条执行路径的 Spec 命令保持一致。

## 界面结构

`分析参数` 展开区域按以下顺序显示：

1. 预设选择和应用按钮
2. 基础参数
3. 高级参数说明与两列勾选式参数
4. 输出选项 `bind` 和 `cutprimer`
5. 命令预览模式

不新增嵌套 expander，避免 Streamlit 布局和状态问题。
