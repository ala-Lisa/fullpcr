# 分析工作台紧凑间距设计

## 目标

修正侧栏品牌副标题换行，并在不缩小字体、输入控件或点击区域的前提下，减少分析工作台中过大的纵向空白。

## 侧栏文案

- 将侧栏品牌副标题从“fullpcr 全库引物评测平台”改为“全库引物评测平台”。
- 保留公司名称、Logo、BOKUN BIO 标识、环境状态和导航结构。

## 工作台间距

- 为分析工作台增加独立的带 key 容器，作为紧凑样式的作用域。
- 仅在该容器内压缩约 25%–30% 的纵向间距。
- 重点减少标题与说明、组件组之间、分隔线前后，以及参数区与快速分析区之间的空白。
- 保持现有字号、输入框高度、按钮大小、帮助图标和控件可点击区域。
- 不改变控件顺序、参数值、状态管理或命令构建逻辑。

## 页面隔离

紧凑样式只能作用于“分析工作台”。“结果总览”和“报告与下载”继续使用现有间距，不允许通过全局垂直间距规则影响这两个页面。

## 测试与验收

- AppTest 断言侧栏不再显示“fullpcr 全库引物评测平台”，并显示“全库引物评测平台”。
- 静态样式测试确认紧凑规则使用分析工作台专属 class，不使用全站垂直间距覆盖。
- 既有工作台控件、基础参数和高级参数测试继续通过。
- 运行 GUI 测试、全量测试、编译检查和 `git diff --check`。

## 范围外

不修改结果总览、报告下载、导航卡片尺寸、字体系统、颜色、输入验证、分析参数或后端命令。
